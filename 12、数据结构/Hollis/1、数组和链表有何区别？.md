数组和链表的区别如下所示：

| 比较    | 数组                                                                                 | 链表                                 |
| ----- | ---------------------------------------------------------------------------------- | ---------------------------------- |
| 内存    | **==连续内存空间==**                                                                     | **==不连续==**，通过指针连接各个元素             |
| 元素大小  | 相等；<br>基本数据类型数组中每个元素占用大小相等的内存，对象引用数组中每个元素都是对象的引用，引用大小也是相等的                         | 不相等                                |
| 访问方式  | 根据索引查找：**==随机访问==**，通过数组地址 + 索引 + 每个元素大小，O(1)<br>根据内存查找：**==顺序访问==**，从头遍历元素内容，O(n) | **==顺序访问==**，必须从头节点开始遍历，时间复杂度，O(n) |
| 插入/删除 | O(n)，需要移动元素                                                                        | O(1)                               |
| 缓存友好型 | 友好；**连续内存==便于CPU缓存预加载==，访问速度快**                                                    | 不友好；**节点分散在内存各处**                  |

# 典型回答

从定义上讲：
数组和链表都是数据的集合。
1. **数组中的元素是连续的，通过下标进行访问**，当我们获取到下标后，就可以随意访问数组中的值
2. **链表中的元素则是不连续的，必须获得链表中某个元素后**，才能顺序访问该元素的周围元素，我们没办法随意访问链表中的元素。链表分为单向链表，双向链表，环形链表等

从实现上来讲：
1. 数组可以**由一块连续区域的内存实现，其中，内存地址可以作为数组的下标，该地址中的值就是数组中元素的值**。因为数组占用的是一块空间，所以数组的大小申请之后就会固定；
2. 链表可以**由不连续的内存存储实现，每个元素都会存储下一个元素的地址**。（如果是双向链表的话，元素则会还会存储上个元素的地址）。因为链表中存储了元素的地址，所以链表可以在内存足够的情况下随意申请空间
