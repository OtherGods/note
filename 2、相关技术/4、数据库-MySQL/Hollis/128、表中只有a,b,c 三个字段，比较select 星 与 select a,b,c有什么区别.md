# 典型回答

这个问题是下面这个问题的扩展：
[124、为什么要尽量避免使用select 星 ？](2、相关技术/4、数据库-MySQL/Hollis/124、为什么要尽量避免使用select%20星%20？.md)

如果，在当前表只有字段 `a, b, c` 时，这两个查询语句的查询结果是一致的。

在性能上差别也不大，虽然有人说用 `select *` 会带来一次额外的字段映射和转换开销，但是当对于只有a,b,c三个字段的情况下，这个开销几乎可以忽略。

**那么如果非要说区别的话，主要是明确性方面，`SELECT a, b, c`明确指定了字段，代码更加清晰，而且他不受后期的表结构（增删字段）变化的影响。而`SELECT *`不够明确， 对表结构的变化，更加敏感**
