断言的概念：断言机制允许在测试代码期间向代码中插入一个检查，而在生产代码中会自动删除这些检查。

java语言引入的assert关键字；这个关键字有两种形式：
1. `assert condition`：`condition`为true正常执行，否则抛出异常`AssertionError`
2. `assert condition ：expression`；`condition`为true正常执行，否则将`expression`传如`AssertionError`构造器，并抛出该异常

示例代码：
```java
import java.io.*;

public class test {
    public static void main(String args[]) {
		assert false : "断言失败";
    }
}
```

反编译后代码：
```java
public class test {
    public test() {}
    public void com(int i) {
        if(!$assertionsDisabled && i == 2)
            throw new AssertionError("断言失败");
        else
            return;
    }
    static final boolean $assertionsDisabled = !test.desiredAssertionStatus();
}
```

在默认情况下，断言是禁用的。可以在运行程序的时候使用 `-enableassertions` 或 `-ea` 选项启用断言：`java -enableassertions MyApp`

也可以在整个包中或者某个类中启用或者禁用断言：
- 启用断言：`java -ea MyClass -ea:com.mycompany.mylib MyApp`：这条命令将`MyClass`类以及`com.mycompany.mylib`包和它的子包中所有类打开断言。选项`-ea`将打开无名包中所有的断言
- 禁用断言：`java -ea:... -da:MyClass MyApp`：也可以使用选项 `-disableassertions` 或者 `-da` 在某个特定类和包中禁用断言。
	也可以通过编程控制类加载器的断言状态

注意：有些类不是由类加载器加载的，而是直接由虚拟机加载的
- 启用和禁用所有断言的`-ea`和`-da`开关不能应用到那些没有类加载器的“系统类”上，对于这些系统类，需要使用 `-enablesystemassertions`、`-esa`开关启用断言
