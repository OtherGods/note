建造者模式是一种创建型的设计模式，它通过将一个复杂对象的构建与它的表示相分离，使得同样的构建过程可以创建不同的表示。

这种模式是用来解决直接创建对象（例如通过直接实例化或者使用构造器）可能导致的复杂性问题。

在 Java 中，建造者模式常常被用来构建具有多个字段的复杂对象，尤其是当对象含有非强制性字段时。

在我们的项目中，对于一些参数的创建，使用了建造者模式，简化构造过程。如ChainProcessResponse：
```java
@Setter  
@Getter  
@ToString  
public class ChainProcessResponse<T> extends BaseResponse {  
    private static final long serialVersionUID = 1L;  
      
    private T data;  
      
    public static class Builder<T> {  
        private T data;  
          
        private String responseCode;  
          
        private String responseMessage;  
          
        public Builder(T data) {  
            this.data = data;  
        }  
          
        public Builder() {  
        }  
          
        public Builder data(T data) {  
            this.data = data;  
            return this;  
        }  
          
        public Builder responseCode(String responseCode) {  
            this.responseCode = responseCode;  
            return this;  
        }  
          
        public Builder responseMessage(String responseMessage) {  
            this.responseMessage = responseMessage;  
            return this;  
        }  
          
        public ChainProcessResponse buildSuccess() {  
            ChainProcessResponse response = new ChainProcessResponse();  
            response.setSuccess(true);  
            response.data = this.data;  
            return response;  
        }  
          
        public ChainProcessResponse buildFailed() {  
            ChainProcessResponse response = new ChainProcessResponse();  
            response.setSuccess(false);  
            response.setResponseCode(this.responseCode);  
            response.setResponseMessage(this.responseMessage);  
            return response;  
        }  
    }  
}
```

在上述代码中，ChainProcessResponse 类使用了建造者（Builder）模式来构建对象

以下是代码示例中建造者模式的几个关键应用点：
1. 链式调用：建造者模式允许通过链式调用（fluent interface）设置属性，使得代码的可读性更高，更易于维护。链式调用是通过在建造者类的每个设置属性的方法中返回建造者对象自身来实现的，从而可以连续调用多个设置方法，每个方法的调用为对象配置了一个属性。
2. 泛型支持：通过使用泛型，ChainProcessResponse.Builder 为任意类型的 data 属性提供了类型安全的构建方式。这意味着无论ChainProcessResponse 需要封装的数据是什么类型，建造者都可以支持它。
3. 分离构建步骤和表示：建造者模式允许构建步骤（比如responseCode和 data的设置）与对象最终表示（ChainProcessResponse的实例）分离。这个模式使得客户端代码可以在不暴露对象内部表示的情况下，进行复杂对象的构建。
4. 不同的构建方法：在示例中，建造者提供了buildSuccess和buildFailed两种方法，它们分别对应构建表示成功和失败的ChainProcessResponse对象。这两个方法封装了不同的构建逻辑，可以根据操作的结果，选择合适的方法来构建响应对象。
5. 保持对象状态的一致性：一旦完成了所有需要的设置，使用buildSuccess或buildFailed方法可以构建一个一致、不变的ChainProcessResponse对象。由于所有需要设置的字段都是通过建造者一步步设置的，这帮助保持对象的一致性，并避免了错误的构建操作。

下面是该模式使用的示例
```java
@Override  
public ChainProcessResponse<ChainCreateData> createAddr(ChainProcessRequest request) {
    return new ChainProcessResponse.Builder().responseCode(ChainCodeEnum.SUCCESS.name()).data(  
            new ChainCreateData(request.getIdentifier(), "mockBlockChainAddr", "mockBlockChainName",  
                    ChainType.MOCK.name())).buildSuccess();  
}
```

在这个例子中，建造者模式允许调用者在不需要创建额外构造器的情况下，以一种直接、清晰地方式来设置ChainProcessResponse所需的各种属性。

一旦属性被设置，调用buildSuccess方法会创建一个新的ChainProcessResponse对象，并将它返回给调用者。如果需要创建失败的响应，可以使用buildFailed方法来代替。这种方式的灵活性使客户端代码变得非常简洁，并且容易理解。
