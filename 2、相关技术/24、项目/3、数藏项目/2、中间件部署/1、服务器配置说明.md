
我们的项目是一个分布式的项目，虽然我们已经尽可能的做了很多工作，让应用的部署和启动尽可能简单，比如我们把多个业务模块都放在了nft-turbo-app 中可以一键启动所有应用。比如我们提供了对第三方的服务的 mock，比如我们支持切换成不依赖 es而直接依赖数据库的方案等等。

但是，它还是依赖了很多中间件，所以想要都部署起来的话，还是需要一会的资源的，主要是内存资源。大致的内存需求如下：

| 部署内容                 | 内存要求 | 是否必须 |
| -------------------- | ---- | ---- |
| nft-turbo-app        | 2G   | Y    |
| nft-turbo-auth       | 1G   | Y    |
| nft-turbo-gateway    | 1G   | Y    |
| Nacos                | 2G   | Y    |
| RocketMQ             | 8G   | N    |
| XXL-JOB              | 2G   | N    |
| ElesticSearch+Kibana | 4G   | N    |
| Canal                | 2G   | N    |
| Sentinel             | 2G   | N    |

总体的话可能需要不低于24G 的内容才能把整个项目完整部署起来的话，这里还不算 MySQL 和 Redis。
