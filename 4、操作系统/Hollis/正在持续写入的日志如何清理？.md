# 典型回答

如果一个日志正在持续写入，但是它的内容太多了，占用了很大的内存，这时候如果想要清理的话，直接删除是不行的，因为一旦删除这个文件就不存在了，应用会因为找不到日志文件而报错。

那么可以通过以下方式清空文件内容：
```shell
# Shell 的重定向操作符 > 在没有前置命令时，会直接打开文件并将其截断为零字节；
# 立即将 `application.log` 文件的大小变为 0 字节，内容完全清空。
> application.log

# 经典的、但略显“过时”的清空文件方法
# /dev/null 是一个特殊的设备文件，它总是返回 "文件结束"，即不输出任何内容。
# 将 `file_name` 清空为零字节
cat /dev/null > file_name

# `echo` 命令会输出一个空字符串。但请注意，默认情况下，`echo` 会在输出的末尾自动添加一个换行符。
# 文件**不是完全空的**！它包含**一个换行符**。在文本文件中，这表现为一个空行。文件大小通常是 1 字节（换行符 `\n`）或 2 字节（在某些系统上可能是 `\r\n`）
echo "">file_name
```

|命令|最终文件内容|文件大小|效率|推荐度|
|---|---|---|---|---|
|`> application.log`|**真正空**|**0 字节**|**最高**|⭐⭐⭐⭐⭐ **（首选）**|
|`cat /dev/null > file_name`|**真正空**|**0 字节**|较高|⭐⭐⭐⭐ （经典，意图明确）|
|`echo "">file_name`|**一个换行符**|**1 或 2 字节**|较高|⭐⭐ （不推荐用于清空，除非需要留一个空行）|
### 不要混淆 `>` 和 `>>`

- `>`：重定向，会**覆盖**目标文件。
- `>>`：重定向，会**追加**到目标文件末尾。
