JavaGuide：

1. [单元测试到底是什么？应该怎么做？](https://javaguide.cn/system-design/basis/unit-test.html)
    

为了保证代码质量，为重构提供信心，我们的项目一般都会写单元测试。

你需要知道如何使用 ：

- [Junit](https://link.zhihu.com/?target=https%3A//junit.org/junit5/) ： 当前最流行的 Java 测试框架！提到 Java 测试，往往离不开它！
    
- [Mockito](https://link.zhihu.com/?target=https%3A//github.com/mockito/mockito) ：用于 Mock 测试中用的类，简答来说就是创建一个虚拟对象。当一个类比较复杂或者和要测试的内容无关的时候，Mock 非常有用！
    
- [PowerMock](https://link.zhihu.com/?target=https%3A//github.com/powermock/powermock) ：同样用于 Mock 测试中用的类，不过， PowerMock 可以实现完成对 private/static/final 方法的 Mock（模拟），而 Mockito 仅仅可以对普通的方法进行 Mock。
    
- ......
    

除了上面介绍到的几个测试相关的框架之外，下面几个也可以学习一下（可选）：

- [Spock](https://link.zhihu.com/?target=https%3A//spockframework.org/) ：Spock 是一个 Java 和 Groovy 应用程序的测试和规范框架。它能够让我们的测试代码更规范，内置多种标签来规范单测代码的语义。相关阅读：[Spock单元测试框架介绍以及在美团优选的实践](https://link.zhihu.com/?target=https%3A//tech.meituan.com/2021/08/06/spock-practice-in-meituan.html)
    
- [AssertJ](https://link.zhihu.com/?target=https%3A//github.com/assertj/assertj-core) ：流式断言神器，可以与 JUnit，TestNG 等其他测试框架一起使用。
